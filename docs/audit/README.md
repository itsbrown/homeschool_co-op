# Payment-Safety Audit Reports

This directory holds **durable audit artifacts** for tasks in the payment-safety tier and any future task whose close gate requires reproducible proof of behavior.

## Why this directory exists

Tasks #219 and #238 both produced proof reports during execution that were lost on merge because they were written under `.local/tasks/` (an agent-private path that is not always merged back to main). Result: governance gap — the audit existed transiently in agent logs but not on disk.

**Rule (effective task #222 onward):** All audit artifacts required by a hard close gate MUST be written under `docs/audit/<task-number>-<slug>-report.md`. They are tracked in git and survive merges.

## Standard report filename

```
docs/audit/<task-number>-<slug>-report.md
```

Examples:
- `docs/audit/219-stop-payments-disappearing-report.md`
- `docs/audit/222-refund-durability-report.md`
- `docs/audit/212-membership-balance-source-report.md`
- `docs/audit/201-reallocator-tests-report.md`

## Optional sibling evidence folder

When a task produces bulky raw evidence (JSON snapshot dumps, multi-file test logs, schema dumps, etc.) that would clutter the report markdown, place those raw files in a sibling folder at:

```
docs/audit/<task-number>-evidence/
```

Examples already in this directory:
- `docs/audit/212-evidence/property*.json` — one raw `calculateCartSnapshot` response per property.
- `docs/audit/201-evidence/canonical-test-files.txt`, `batch-preexisting-unit.log`, `git-sha-pin.txt` — raw test enumeration + run logs + SHA capture.

The main `*-report.md` MUST still contain inline raw output for every required property (per the must-pass clause). The evidence folder is a supplement for full-fidelity artifacts, not a replacement.

## Universal must-pass clause

Every report in this directory must satisfy:

> **Task cannot close without raw command output, raw SQL/query results (if applicable), and raw API/test responses captured in the report file under `docs/audit/`.** Paraphrased summaries, "test passed" assertions without captured output, or omitted evidence in any required section all block closure.

## Required header (every report)

```
# <Task Title> — Audit Report

- **Task ref:** #NNN
- **Code SHA verified:** <git rev-parse HEAD>
- **Commit:** <git log -1 --format='%H %ci %s' HEAD>
- **Working tree:** clean / dirty (paste `git status -s` if dirty)
- **Report generated:** <ISO timestamp>
- **Generated by:** <task-agent or human, with id/email>
```
