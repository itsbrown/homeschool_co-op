Step 1: Analyze the Issue
1.1: Context of the Query
The query in question is:

sql



SELECT rolname, rolconfig
FROM pg_roles
WHERE rolname = 'authenticator';
Purpose: This query checks the authenticator role's configuration (specifically the search_path) to confirm that custom schemas (users, schools, etc.) are exposed to PostgREST, which the Supabase JavaScript client uses to access the database.
Expected Output:
text

Error: "can't find source pg_roles" indicates that the pg_roles table is not accessible in the context where the query is being executed.
1.2: What is pg_roles?
pg_roles: This is a system catalog table in PostgreSQL that stores information about database roles (e.g., authenticator, postgres, anon). It includes columns like rolname (role name) and rolconfig (configuration settings, such as search_path).
Access: pg_roles is part of PostgreSQL's system catalog and should be accessible to users with sufficient privileges, such as the postgres superuser role. However:
Regular users may have limited access to system catalogs.
In Supabase, the SQL Editor typically runs queries as the postgres role (a superuser), which should have full access to pg_roles.
1.3: Possible Causes of the Error
Supabase Restrictions:
Supabase may restrict access to certain system catalog tables like pg_roles in the SQL Editor to prevent users from modifying sensitive database configurations, even when running as the postgres role.
While the postgres role has access to pg_roles, Supabase might block queries to system catalogs for security reasons.
Incorrect Schema or Search Path:
The query assumes pg_roles is in the default search_path. If the search_path is misconfigured (e.g., doesn’t include pg_catalog), the table might not be found.
However, pg_roles is in the pg_catalog schema, which is always included in the default search_path unless explicitly altered.
Typographical Error:
The query might have a typo, or you might be querying a different table (e.g., pg_role instead of pg_roles).
The correct table name is pg_roles, so this is unlikely unless the query was modified.
Database Context:
If you’re running the query in a different database or context (e.g., not on the Supabase instance), pg_roles might not exist or be accessible.
In Supabase, you’re querying the PostgreSQL database hosted by Supabase, so pg_roles should exist.

javascript




router.get("/my-school", async (req, res) => {
  try {
    console.log('🏫 Fetching school data for admin');
    
    const authHeader = req.headers.authorization;
    if (!authHeader) {
      console.log('❌ No authorization header provided');
      return res.status(401).json({ message: "No authorization header" });
    }

    const token = authHeader.replace('Bearer ', '');
    console.log('🔒 Token:', token);
    
    const { createClient } = await import('@supabase/supabase-js');
    if (!process.env.SUPABASE_URL || !process.env.SUPABASE_ANON_KEY) {
      console.log('❌ Supabase configuration missing');
      throw new Error('Supabase configuration missing');
    }
    const supabase = createClient(
      process.env.SUPABASE_URL,
      process.env.SUPABASE_ANON_KEY,
      {
        global: {
          headers: {
            Authorization: `Bearer ${token}`
          }
        }
      }
    );
    
    console.log('🔍 Verifying token with Supabase Auth...');
    const { data: { user }, error: authError } = await supabase.auth.getUser(token);
    
    if (authError || !user) {
      console.error('❌ Auth error:', authError?.message, authError?.stack);
      return res.status(401).json({ message: "Invalid token", error: authError?.message });
    }

    console.log('✅ Authenticated user:', user.email);

    console.log('🔍 Querying public.accounts for email:', user.email);
    const { data: userData, error: userError } = await supabase
      .from('accounts')
      .select('id')
      .eq('email', user.email)
      .eq('role', 'school_admin')
      .single();

    if (userError || !userData) {
      console.error('❌ User lookup error:', userError?.message);
      return res.status(404).json({ 
        message: "School admin user not found",
        error: userError?.message
      });
    }

    console.log('✅ Found user ID:', userData.id);

    console.log('🔍 Querying public.schools for created_by:', userData.id);
    const { data: schoolData, error: schoolError } = await supabase
      .from('schools')
      .select('*')
      .eq('created_by', userData.id)
      .single();

    if (schoolError || !schoolData) {
      console.error('❌ School lookup error:', schoolError?.message);
      return res.status(404).json({ 
        message: "No school found for this administrator",
        error: schoolError?.message
      });
    }

    console.log('🚀 Returning school data from database:', schoolData.name);
    res.json(schoolData);
  } catch (error) {
    console.error("❌ Error fetching school information:", error.message, error.stack);
    res.status(500).json({ message: "Error fetching school information", error: error.message });
  }
});
