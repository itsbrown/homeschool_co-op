Replit, build a new admin-only report: "Family Retention Report" (place under Reports > Enrollment or as a new "Retention" tab/section in admin dashboard).

Goal: Let admins compare enrollments between two periods/sessions and see returning vs new vs dropped families. This measures retention (e.g., % of parents from last session who re-enrolled children this session) — high-value for planning/marketing/teacher allocation.

This report should follow patterns from existing financial reports: summary cards at top, filterable table, CSV export, reuse existing components (tables, date pickers, export utils, school validation).

Inputs / Filters (top of page):
- Period 1 (Previous Session): date range picker (start/end) or session dropdown if sessions are modeled
- Period 2 (Current Session): date range picker (start/end) or session dropdown
- School: dropdown of admin's accessible schools (validate ownership)

Backend: New endpoint /api/admin/retention/compare (GET)
Parameters: period1Start, period1End, period2Start, period2End (ISO dates), schoolId (required/filtered)

Logic (reuse storage methods like getProgramEnrollmentsByDateRange, getProgramEnrollmentsByParent, etc. — targeted queries only, no full-table scans):
1. Fetch Period 1 enrollments (non-terminal status, within date range, school filter) → unique set of family keys (parentId or parent.email as primary; optionally include childId for per-child detail)
2. Fetch Period 2 enrollments (same filters) → unique family keys
3. Compute sets:
   - Returning: intersection (in both periods)
   - New: in Period 2 but not Period 1
   - Dropped: in Period 1 but not Period 2
4. For detailed rows (union of both periods):
   - parent email/name
   - child name(s) — comma-separated if multiple
   - Period 1: Yes/No + class names (if Yes)
   - Period 2: Yes/No + class names (if Yes)
   - Status: "Returning", "New", "Dropped"
5. Summary:
   - returningCount, returningPercentage = (returning / period1UniqueFamilies) * 100
   - newCount
   - droppedCount
   - Optional note: "vs NAIS median retention ~92%" (hardcoded or config)
6. Return: { summary: {...}, rows: [...] }

Frontend: client/src/pages/admin/RetentionReport.tsx (or tab in existing reports page)
- Filters form (date pickers or dropdowns)

  - Returning Families: X (Y%)
  - New Families: Z

- Sortable/filterable table with columns: Parent, Child(ren), Period 1 Enrolled, Period 2 Enrolled, Status
