Approach to Train the AI for Worksheets, Coloring Pages, Crossword Puzzles, etc.
1. Data Preparation for Training
Types of Materials:
Worksheets: Structured documents with questions or exercises (e.g., math problems, fill-in-the-blank sentences).
Coloring Pages: Black-and-white line art with clear outlines (already implemented in Sprint 4, Task 4.5).
Crossword Puzzles: Grids with clues and answers, tailored to subjects (e.g., vocabulary words for Tycoons).
Other Materials: Includes word searches, mazes, or matching exercises, depending on age and subject.
Data Sources:
Public Domain and Creative Commons: Use educational resources from sites like Teachers Pay Teachers (with permission), Open Educational Resources (OER), or public domain repositories (e.g., Library of Congress).
Open-Source Datasets: Datasets like those on Kaggle containing educational worksheets, puzzles, and activities (e.g., “Math Worksheet Dataset”).
Custom-Created Content: Commission or crowdsource sample worksheets, coloring pages, and puzzles from educators in your community (e.g., ASA teachers), ensuring they align with your curriculum (e.g., subtraction for Tycoons).
Knowledge Base: Leverage the platform’s existing knowledge base (KnowledgeBase schema) to extract relevant content (e.g., math problems, vocabulary words) for generating materials.
Preprocessing:
Worksheets: Extract text (questions, answers) and structure (e.g., layout, question types) from sample worksheets, converting them into a structured format (e.g., JSON: { "question": "2 + 3 = ?", "answer": "5" }).
Coloring Pages: Already vectorized (Sprint 2, Task 2.7), but expand the dataset with more diverse themes (e.g., math symbols, historical figures).
Crossword Puzzles: Parse sample puzzles into grids, clues, and answers (e.g., { "grid": [["A", "B", "C"]], "clues": [{"across": "1. A fruit (5 letters)", "answer": "APPLE"}] }).
Storage: Store preprocessed data in MongoDB as a new collection (ActivityTemplates) for training.
2. Train the AI Model
Model Selection:
Base Model: Use Claude-3.7-Sonnet (already integrated) for text generation (e.g., worksheet questions, crossword clues).
Specialized Model: Fine-tune a transformer model like distilbert-base-uncased for structured content generation (e.g., worksheets, puzzles), focusing on formatting and logic (e.g., crossword grid generation).
Coloring Pages: Reuse the existing coloring page generation pipeline (Sprint 4, Task 4.5), expanding its training data.
Training Process:
Worksheets:
Fine-tune the model on preprocessed worksheet data to generate questions and answers based on subject and difficulty (e.g., “Generate a subtraction worksheet for ages 6-7”).
Train on question types (e.g., multiple-choice, fill-in-the-blank) and layouts (e.g., 5 questions per page with space for answers).
Crossword Puzzles:
Train the model to generate grids, clues, and answers using crossword datasets.
Use a rule-based approach for grid generation (e.g., ensure words intersect correctly) and fine-tune for clue generation (e.g., “A fruit (5 letters)”).
Coloring Pages:
Expand the existing training data with more diverse themes (e.g., math symbols for Tycoons math lessons).
Fine-tune the model to ensure age-appropriate complexity (e.g., simple shapes for ages 6-7, detailed scenes for ages 11-18).
Other Materials:
Train on word search datasets (e.g., generate a 10x10 grid with 5 vocabulary words).
Use rule-based logic for mazes and matching exercises, fine-tuning for content generation (e.g., matching animals to habitats).
Training Data:
Combine public domain, open-source, and custom-created datasets to ensure diversity and relevance.
Augment with knowledge base content to align with the platform’s curricula (e.g., use subtraction concepts from the knowledge base for Tycoons worksheets).
3. Content Generation and Integration
Generation Pipeline:
Input: The AI takes parameters like age, subject, difficulty, and activityType (e.g., “worksheet”, “crossword”).
Process:
For worksheets: Generate questions and answers, format into a printable layout (e.g., PDF using pdfkit).
For coloring pages: Generate black-and-white line art, ensuring clear outlines and age-appropriate complexity.
For crossword puzzles: Generate a grid, place words, and create clues, outputting a printable format.
For other materials: Generate content based on trained models (e.g., word search grid, maze paths).
Output: Save generated materials as PDFs in AWS S3, with metadata stored in MongoDB (e.g., Activity schema: type, url, subject, ageRange).
Integration:

Child Profiles: Add generated materials to child profiles as recommended activities or homework (e.g., “Complete Subtraction Worksheet”).

Parent Dashboard: Parents can view and download materials from their child’s profile (e.g., “Download Subtraction Worksheet”).
4. Integration with Existing Features
AI Assistant: Can recommend and provide links to generated materials (e.g., “Emma might benefit from this subtraction worksheet: [download link]”).
Knowledge Base: Used as a source for content (e.g., math problems, vocabulary words) to ensure alignment with curricula.

import json
import torch
from transformers import DistilBertTokenizer, DistilBertForSequenceClassification, Trainer, TrainingArguments

tokenizer = DistilBertTokenizer.from_pretrained('distilbert-base-uncased')

def train_activities():
    worksheets = json.load(open('activity_data/worksheets.json'))
    crosswords = json.load(open('activity_data/crosswords.json'))
    # Simplified training example for worksheets
    inputs = [w['questions'][0]['question'] for w in worksheets]
    labels = [int(w['questions'][0]['answer']) for w in worksheets]  # Simplified label
    encodings = tokenizer(inputs, truncation=True, padding=True, return_tensors='pt')
    dataset = torch.utils.data.TensorDataset(encodings['input_ids'], encodings['attention_mask'], torch.tensor(labels))
    model = DistilBertForSequenceClassification.from_pretrained('distilbert-base-uncased', num_labels=10)
    training_args = TrainingArguments(
        output_dir='./activity_model',
        num_train_epochs=3,
        per_device_train_batch_size=8,
        save_steps=10_000,
        save_total_limit=2,
    )
    trainer = Trainer(model=model, args=training_args, train_dataset=dataset)
    trainer.train()
    model.save_pretrained('activity_model')

if __name__ == '__main__':
    train_activities()

const express = require('express');
const { spawn } = require('child_process');
const path = require('path');
const PDFDocument = require('pdfkit');
const AWS = require('aws-sdk');
const s3 = new AWS.S3();
const router = express.Router();

router.post('/generate', async (req, res) => {
  const { age, subject, activityType } = req.body;
  const python = spawn('python', [path.join(__dirname, '../../../ai/src/generate_activities.py'), JSON.stringify({ age, subject, activityType })]);
  let output = '';
  python.stdout.on('data', (data) => output += data);
  python.stderr.on('data', (data) => res.status(500).send(data.toString()));
  python.on('close', async () => {
    const activity = JSON.parse(output);
    const doc = new PDFDocument();
    let buffers = [];
    doc.on('data', buffers.push.bind(buffers));
    doc.on('end', async () => {
      const pdfData = Buffer.concat(buffers);
      const key = `activities/${activityType}-${Date.now()}.pdf`;
      await s3.upload({
        Bucket: process.env.AWS_S3_BUCKET,
        Key: key,
        Body: pdfData,
        ContentType: 'application/pdf',
      }).promise();
      const downloadLink = s3.getSignedUrl('getObject', {
        Bucket: process.env.AWS_S3_BUCKET,
        Key: key,
        Expires: 3600,
      });
      res.json({ downloadLink });
    });
    doc.fontSize(16).text(`${activityType} for ${subject}`, { align: 'center' });
    doc.moveDown();
    if (activityType === 'worksheet') {
      activity.questions.forEach(q => {
        doc.fontSize(12).text(`${q.question} (Answer: ${q.answer})`);
        doc.moveDown();
      });
    } else if (activityType === 'crossword') {
      doc.text('Crossword Puzzle');
      activity.clues.forEach(clue => doc.text(clue.across));
    }
    doc.end();
  });
});

module.exports = router;
