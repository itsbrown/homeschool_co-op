I understand the issues you're facing with the class details persistence in the Adaptive AI-Driven Curriculum Generation and Learning Management System, specifically on the Class List Page (SimpleClassesPage). The problems are:
1. Date Changes by One Day: The date displayed on the Class List Page is off by one day from the date entered during class creation or editing.
2. Price Gets Two Extra Zeros: The price displayed has an additional multiplication by 100 (e.g., $150 becomes $15000), due to mishandling of the price conversion between dollars and cents.
These issues are affecting the user experience for managing classes like Tycoons (ages 6-7) at American Seekers Academy (ASA), where accurate date and price display are critical. Let’s analyze the root causes and implement fixes to ensure class details persist correctly across views.
________________


Analysis of the Issues
1. Date Changes by One Day
* Problem: When a class is created or edited (e.g., with a startDate of "2025-04-07"), the Class List Page displays the date as one day earlier or later (e.g., "2025-04-06" or "2025-04-08").
* Cause:
   * Timezone Mismatch: The dates are stored in the database as Date objects, which are typically in UTC. When the Class List Page formats the date using the browser's date formatter (e.g., toLocaleDateString), it converts the UTC date to the local timezone (e.g., EDT, which is UTC-4 on May 17, 2025). This conversion can shift the date by one day if the UTC date crosses midnight in the local timezone.
   * Date Parsing: The startDate and endDate fields in the form (e.g., "2025-04-07") are likely interpreted as local dates by the browser, but when saved to the database, they might be converted to UTC without proper timezone handling, leading to a discrepancy.
* Impact: Users see incorrect dates on the Class List Page, which can cause confusion when managing class schedules.
2. Price Gets Two Extra Zeros
* Problem: The price entered as $150 is displayed as $15000 on the Class List Page due to multiple multiplications by 100.
* Cause:
   * Multiple Price Conversions:
      * During class creation/editing, the server multiplies the price by 100 to store it in cents (e.g., $150 → 15000 cents).
      * When displaying on the Class List Page, the price is supposed to be divided by 100 to convert back to dollars (15000 cents → $150). However, the system fails to check if the price is already in cents, leading to an additional multiplication by 100 (15000 → 1500000 cents), which is then divided by 100 to display $15000.
   * Lack of Price State Tracking: The system doesn’t track whether the price is in dollars or cents at each step, causing redundant conversions.
* Impact: Incorrect pricing (e.g., $15000 instead of $150) misleads users and could affect enrollment decisions or financial calculations for ASA classes.
________________


Solution: Fix Date and Price Issues
1. Fix Date Changes by One Day
To ensure dates are consistent across views:
* Store Dates in UTC with Explicit Timezone Handling: When saving dates to the database, ensure they are interpreted as UTC dates and stored accordingly.
* Display Dates with Consistent Timezone: When displaying dates on the Class List Page, format them as UTC dates to avoid browser timezone adjustments, or explicitly adjust for the user’s timezone if needed.
* Use a Date Library: Use a library like date-fns to handle date formatting consistently across the application.
2. Fix Price Multiplication Issue
To prevent redundant price conversions:
* Track Price State: Add a flag or metadata to indicate whether the price is stored in cents or dollars, or ensure the price is only converted once during saving.
* Normalize Price on Retrieval: When retrieving the price from the database, ensure it’s consistently treated as cents and converted to dollars only once for display.
* Remove Redundant Conversions: Update the server and client logic to avoid multiple multiplications by 100.

const express = require('express');
const Product = require('../models/product');
const router = express.Router();


// Create a new class
router.post('/classes', auth(['admin']), async (req, res) => {
  const { programType, title, description, targetAge, capacity, startDate, endDate, daysOfWeek, timeOfDay, pricingModel } = req.body;


  try {
    // Ensure dates are stored as UTC
    const start = new Date(startDate + 'T00:00:00Z'); // Explicitly set to UTC
    const end = new Date(endDate + 'T00:00:00Z');     // Explicitly set to UTC


    // Process price: Convert to cents only if not already in cents
    const priceInCents = pricingModel.price * 100; // Assuming price is in dollars


    const newClass = new Product({
      programType,
      title,
      description,
      targetAge,
      capacity,
      startDate: start,
      endDate: end,
      daysOfWeek,
      timeOfDay,
      pricingModel: { ...pricingModel, price: priceInCents, inCents: true }, // Add flag to indicate price is in cents
    });


    await newClass.save();
    res.json(newClass);
  } catch (err) {
    console.error('Error creating class:', err);
    res.status(500).json({ message: 'Failed to create class' });
  }
});


// Update an existing class
router.patch('/classes/:id', auth(['admin']), async (req, res) => {
  const { id } = req.params;
  const { programType, title, description, targetAge, capacity, startDate, endDate, daysOfWeek, timeOfDay, pricingModel } = req.body;


  try {
    const existingClass = await Product.findById(id);
    if (!existingClass) {
      return res.status(404).json({ message: 'Class not found' });
    }


    // Update dates as UTC
    if (startDate) existingClass.startDate = new Date(startDate + 'T00:00:00Z');
    if (endDate) existingClass.endDate = new Date(endDate + 'T00:00:00Z');


    // Update price: Convert to cents only if not already in cents
    if (pricingModel && pricingModel.price) {
      existingClass.pricingModel.price = pricingModel.price * 100; // Assuming price is in dollars
      existingClass.pricingModel.inCents = true;
    }


    existingClass.programType = programType || existingClass.programType;
    existingClass.title = title || existingClass.title;
    existingClass.description = description || existingClass.description;
    existingClass.targetAge = targetAge || existingClass.targetAge;
    existingClass.capacity = capacity || existingClass.capacity;
    existingClass.daysOfWeek = daysOfWeek || existingClass.daysOfWeek;
    existingClass.timeOfDay = timeOfDay || existingClass.timeOfDay;


    await existingClass.save();
    res.json(existingClass);
  } catch (err) {
    console.error('Error updating class:', err);
    res.status(500).json({ message: 'Failed to update class' });
  }
});


// Fetch all classes
router.get('/classes', async (req, res) => {
  try {
    const classes = await Product.find({ programType: 'class' });
    // Convert price back to dollars for display
    const formattedClasses = classes.map(cls => ({
      ...cls._doc,
      startDate: cls.startDate.toISOString().split('T')[0], // Format as YYYY-MM-DD in UTC
      endDate: cls.endDate.toISOString().split('T')[0],     // Format as YYYY-MM-DD in UTC
      pricingModel: {
        ...cls.pricingModel,
        price: cls.pricingModel.inCents ? cls.pricingModel.price / 100 : cls.pricingModel.price, // Convert to dollars if stored in cents
        inCents: false
      }
    }));
    res.json(formattedClasses);
  } catch (err) {
    console.error('Error fetching classes:', err);
    res.status(500).json({ message: 'Failed to fetch classes' });
  }
});


// Fetch a single class by ID

        ...cls.pricingModel,
        price: cls.pricingModel.inCents ? cls.pricingModel.price / 100 : cls.pricingModel.price, // Convert to dollars
        inCents: false
      }
    };
    res.json(formattedClass);
  } catch (err) {
    console.error('Error fetching class:', err);
    res.status(500).json({ message: 'Failed to fetch class' });
  }
});

import React, { useState, useEffect } from 'react';
import axios from 'axios';
import Card from '../components/Card';
import Button from '../components/Button';
import { useParams, useNavigate } from 'react-router-dom';


const ClassCreationForm = () => {
  const { id } = useParams();
  const navigate = useNavigate();
  const [form, setForm] = useState({
    programType: 'class',
    title: '',
    description: '',
    targetAge: '',
    capacity: 20,
    startDate: '',
    endDate: '',
    daysOfWeek: [],
    timeOfDay: '',
    pricingModel: { type: 'flat', price: 0 },
  });


  useEffect(() => {
    if (id) {
      // Fetch existing class data for editing
      axios.get(`http://localhost:3000/api/admin-classes/classes/${id}`, {
        headers: { Authorization: `Bearer ${localStorage.getItem('token')}` }
      }).then(res => {
        setForm(res.data);
      }).catch(err => {
        console.error('Error fetching class:', err);
      });
    }
  }, [id]);


  const handleSubmit = async () => {
    try {
      if (id) {
        // Update existing class
        await axios.patch(`http://localhost:3000/api/admin-classes/classes/${id}`, form, {
          headers: { Authorization: `Bearer ${localStorage.getItem('token')}` }
        });
      } else {
        // Create new class
        await axios.post('http://localhost:3000/api/admin-classes/classes', form, {
          headers: { Authorization: `Bearer ${localStorage.getItem('token')}` }
        });
      }
      navigate('/classes');
    } catch (err) {
      console.error('Error submitting form:', err);
    }
  };


  return (
    <div className="min-h-screen bg-secondary p-8">
      <h1 className="text-3xl font-bold text-primary mb-4">{id ? 'Edit Class' : 'Create Class'}</h1>
      <Card className="p-4">
        <input type="text" placeholder="Title" value={form.title} onChange={(e) => setForm({ ...form, title: e.target.value })} className="border p-2 w-full mb-2" />
        <textarea placeholder="Description" value={form.description} onChange={(e) => setForm({ ...form, description: e.target.value })} className="border p-2 w-full mb-2" />
        <input type="text" placeholder="Target Age/Grade" value={form.targetAge} onChange={(e) => setForm({ ...form, targetAge: e.target.value })} className="border p-2 w-full mb-2" />
        <input type="number" placeholder="Capacity" value={form.capacity} onChange={(e) => setForm({ ...form, capacity: e.target.value })} className="border p-2 w-full mb-2" />
        <input type="date" placeholder="Start Date" value={form.startDate} onChange={(e) => setForm({ ...form, startDate: e.target.value })} className="border p-2 w-full mb-2" />
        <input type="date" placeholder="End Date" value={form.endDate} onChange={(e) => setForm({ ...form, endDate: e.target.value })} className="border p-2 w-full mb-2" />
        <input type="text" placeholder="Days of Week (comma-separated)" value={form.daysOfWeek.join(',')} onChange={(e) => setForm({ ...form, daysOfWeek: e.target.value.split(',') })} className="border p-2 w-full mb-2" />
        <input type="text" placeholder="Time of Day (e.g., 09:00-12:00)" value={form.timeOfDay} onChange={(e) => setForm({ ...form, timeOfDay: e.target.value })} className="border p-2 w-full mb-2" />
        <select value={form.pricingModel.type} onChange={(e) => setForm({ ...form, pricingModel: { ...form.pricingModel, type: e.target.value } })} className="border p-2 mb-2">
          <option value="flat">Flat Fee</option>
          <option value="per-session">Per Session</option>
          <option value="subscription">Subscription</option>
          <option value="free">Free</option>
        </select>
        <input type="number" placeholder="Price (in dollars)" value={form.pricingModel.price} onChange={(e) => setForm({ ...form, pricingModel: { ...form.pricingModel, price: parseFloat(e.target.value) } })} className="border p-2 w-full mb-2" />
        <Button onClick={handleSubmit}>{id ? 'Update Class' : 'Create Class'}</Button>
      </Card>
    </div>
  );
};
