General Best Practices to Prevent Future Drift

Prop drilling / context rule: Any component displaying money gets it via props/context from a single source (API response or snapshot). Never recalculate from primitives.
Type safety: Use branded types (e.g. type Cents = number & { __brand: 'cents' }) to catch accidental float ops.
E2E tests: Add Cypress/Playwright test that adds item → goes to billing → pays → verifies balance updated correctly (server round-trip).
Monitoring: Log client/server discrepancies in production (Sentry) when snapshot loads.
