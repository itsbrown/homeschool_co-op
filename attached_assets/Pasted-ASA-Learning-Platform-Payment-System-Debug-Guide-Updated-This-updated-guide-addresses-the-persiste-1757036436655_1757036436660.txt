Updated Issue Summary

Core Problem: Payments succeed on Stripe but fail to update enrollment balances, payment history, or UI (billing page banner). For monthly plans, the system must handle 3 installments ($1,400 total ≈ $466.67 x 3), applying the first immediately and tracking remaining due.
Root Causes (Potential): Incomplete installment logic in processBalancePayment; webhook failures; inadequate UI refresh; missing payment plan tracking in data files.
Best Practices Integrated:
Payment UX: Clear pricing, real-time validation, specific button labels (e.g., "Pay Now"), minimal fields, post-payment confirmation with updated balance/next steps (from sources like GoCardless, Baymard, Stripe).
Installments: Transparent terms upfront; use Stripe webhooks for updates; track plans manually in backend (since file-based); show next due dates; avoid upfront full payment requests (from Stripe docs and guides).
